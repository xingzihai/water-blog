# 个人网站改动计划

## 目标

将 abstract-portfolio v0.25 改成冰水的真实个人网站，保持美学风格，加入真实元素。

---

## 改动清单

### 1. Gallery区 → 作品展示区

**现状**：9张抽象美学卡片（色彩、文字、光影、节奏、呼吸、涟漪、轨迹、波浪、脉动）

**改成**：真实项目卡片（保持美学风格）

**展示项目**（基于冰水的实际工作）：
| 项目 | 类型 | 卡片内容 |
|------|------|----------|
| TeamVerse | 团队协作网站 | Vue 3 + Vite + 莫兰迪配色系统 |
| particle-grid-scatter | 粒子物理模拟 | Canvas 2D + 椭圆动态效果 |
| aesthetic-portfolio | 个人美学网站 | 印象派美学 + 取景框交互 |
| ESP32灌溉系统 | 硬件项目 | MicroPython + MQTT + 传感器 |
| dot-distortion-shader | WebGL波纹效果 | 物理模拟 + GPU渲染 |

**卡片结构**：
- 项目名称
- 简短描述（一句话）
- 技术栈标签
- 链接（GitHub / Demo）

**验收标准**：
- 卡片保持美学风格（莫兰迪配色、优雅字体）
- 内容真实，链接有效
- 响应式布局正常

---

### 2. Skills区 → 技术栈填充

**现状**：4张占位符卡片（Frontend/Backend/DevOps/Design），技术栈是通用列表

**改成**：真实技术栈分类

| 分类 | 真实技术栈 |
|------|-----------|
| Frontend | Vue 3, Vite, JavaScript, CSS Animation, Canvas |
| Backend | Python, MQTT, REST API |
| Hardware | ESP32, MicroPython, Raspberry Pi |
| Design | 印象派美学, 莫兰迪配色, 动态视觉效果 |

**验收标准**：
- 技术栈真实反映冰水的能力
- 分类合理，标签清晰

---

### 3. Footer区 → 联系方式补充

**现状**：只有GitHub链接

**改成**：添加更多联系方式

- GitHub（已有）
- Email（冰水愿意公开的邮箱）
- 其他社交链接（如有）

**验收标准**：
- 链接有效
- 保持美学风格

---

### 4. 移动端导航菜单 → 功能修复

**现状**：`nav-toggle` 按钮存在，但点击无反应（JS缺失）

**改成**：添加展开逻辑

**验收标准**：
- 点击按钮展开菜单
- 菜单项可点击跳转
- 动画流畅

---

### 5. Hero区 → 微调（可选）

**现状**：xingzihai / 冰水泡枸杞 / 星光落入冰水，温暖慢慢渗透

**建议**：可保持不变，或微调标语（冰水决定）

## 不改动的内容（美学核心）

- 双世界概念（取景框 + 背后世界）
- 莫兰迪/莫奈配色系统
- 六棱柱头像动画
- 背景光影流动
- 萤火虫粒子
- 字体组合（Cormorant Garamond + Inter）
- Philosophy区（印象派理念）

---

## 执行顺序

1. Gallery区内容替换（工作量最大，优先处理）
2. Skills区技术栈填充
3. Footer区联系方式补充
4. 移动端导航菜单修复
