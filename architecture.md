## 一、用户意图探索

### 1.1 核心目标

- **展示美学能力**：不是信息型网站，而是视觉体验型网站
- **舒适交互**：用户停留时感到愉悦，愿意慢下来欣赏
- **配色美观**：已完成的莫兰迪色板系统
- **契合心理学**：遵循认知负荷、格式塔等原则

### 1.2 目标用户画像

| 类型 | 需求 | 预期体验 |
|------|------|----------|
| 潜在雇主/客户 | 快速了解能力 | 3秒内有印象，30秒内有结论 |
| 设计同行 | 学习参考 | 发现设计细节，愿意停留 |
| 普通访客 | 好奇浏览 | 感受到"这个网站很特别" |

### 1.3 成功指标

- **停留时间** > 60秒（慢体验目标）
- **滚动深度** > 80%（内容吸引）
- **视觉冲击**：首屏 3 秒内形成印象
- **细节发现**：用户会注意到微交互

## 三、技术架构设计

### 3.1 技术选型决策

| 维度 | 方案 | 决策理由 |
|------|------|----------|
| **渲染方式** | 纯静态 HTML/CSS | 展示型网站，无动态数据需求，性能最优 |
| **JS 框架** | 原生 JS（无框架） | 展示美学能力，代码简洁可控，无依赖 |
| **CSS 方案** | CSS 变量 + 原生 | 已有完整变量系统，无需预处理器 |
| **构建工具** | 无构建 | 单页静态，直接部署 Cloudflare Pages |
| **字体方案** | Google Fonts CDN | 高质量字体，无需本地托管 |

### 3.2 架构原则

```
1. 性能优先 ──────── 首屏 < 1s，总加载 < 2s
   ├── 无外部 JS 库依赖
   ├── CSS 内联关键样式
   └── 图片懒加载 + placeholder
   
2. 可维护性 ──────── 文件结构清晰
   ├── 单文件组件化思维（CSS 模块化）
   ├── 命名语义化（.hero, .palette-section）
   └── 变量集中管理
   
3. 可扩展性 ──────── 后续可平滑升级
   ├── 结构支持多页面扩展
   ├── CSS 变量支持主题切换
   ├── 数据驱动预留接口
   
4. 部署友好 ──────── Cloudflare Pages 优化
   ├── 无构建步骤
   ├── 静态资源 CDN 自动
   ├── 预渲染友好
```

## 四、交互设计规范

### 4.1 滚动体验设计

**滚动触发动画**：
- Hero 区：进入视口 → 标题渐显（0.6s）
- 各区块：进入视口 → 内容上浮（0.4s stagger）
- 卡片：悬停 → 微上浮 + 阴影加深

**滚动进度指示**：
- 顶部固定进度条（2px 高度）
- 颜色随区块变化（主题色渐变）

**滚动速度感知**：
- `scroll-behavior: smooth`
- 区块间留白充足（避免跳跃感）

### 4.2 微交互设计

| 元素 | 触发 | 效果 | 时长 | 缓动 |
|------|------|------|------|------|
| 按钮 | hover | 上浮 2px + 阴影 | 200ms | ease-out |
| 按钮 | active | 下沉 1px | 100ms | linear |
| 卡片 | hover | 上浮 4px + 阴影 | 300ms | ease-out |
| 链接 | hover | 颜色过渡 | 200ms | smooth |
| 图片 | hover | 缩放 1.02 | 400ms | ease-out |
| 导航 | hover | 背景渐显 | 200ms | smooth |

### 4.3 视觉引导设计

**注意力路径**：
```
Hero 标题（锚点）
    ↓
副标题/简介
    ↓
视觉元素（动态）
    ↓
自动滚动引导 ↓
    ↓
美学展示区（核心）
```

**视觉锚点**：
- Hero 区标题：48px 衬线字体，高对比度
- 动态视觉：微动画吸引注意力
- 卡片悬浮：引导交互探索

### 4.4 响应式策略

**断点设计**：

| 断点 | 宽度 | 布局变化 |
|------|------|----------|
| xs | <480px | 单列，标题缩小 |
| sm | 480-768px | 单列，卡片全宽 |
| md | 768-1024px | 双列，导航展开 |
| lg | 1024-1280px | 标准布局 |
| xl | >1280px | 最大宽度限制 1400px |

**移动端特殊处理**：
- 触控目标 ≥ 44px
- 导航收起为汉堡菜单
- 滚动动画简化（减少 GPU 消耗）
- 图片尺寸自适应

## 五、组件设计规范

### 5.1 Hero 区组件

```html
<section class="hero">
  <div class="hero-content">
    <h1 class="hero-title">...</h1>
    <p class="hero-subtitle">...</p>
    <div class="hero-visual">...</div>
  </div>
</section>
```

**视觉规格**：
- 高度：100vh（首屏全占）
- 标题：56px 衬线，letter-spacing 0.03em
- 副标题：20px 无衬线，color: var(--text-400)
- 背景：渐变（bg-100 → bg-300）
- 动态元素：可选 Canvas/CSS 动画

### 5.2 美学展示组件

```html
<section class="palette-section">
  <header class="section-header">
    <h2 class="section-title">...</h2>
  </header>
  <div class="color-grid">
    <article class="color-card">...</article>
  </div>
</section>
```

**视觉规格**：
- 区块间距：80px
- 标题：32px 衬线
- 卡片：圆角 12px，阴影 shadow-sm
- 卡片悬停：上浮 4px，阴影 shadow-lg

### 5.3 技术能力卡片

```html
<article class="skill-card">
  <header class="skill-card-header">
    <span class="skill-icon">...</span>
    <h3 class="skill-title">...</h3>
  </header>
  <div class="skill-body">...</div>
  <footer class="skill-footer">
    <span class="skill-tag">...</span>
  </footer>
</article>
```

### 5.4 导航组件

```html
<nav class="site-nav">
  <a class="nav-link active">首页</a>
  <a class="nav-link">作品</a>
  <a class="nav-link">关于</a>
  <a class="nav-link">联系</a>
</nav>
```

**交互规格**：
- 固定顶部：sticky
- 背景：半透明模糊（backdrop-filter）
- 悬停：背景渐显 + 文字色变化
- 当前页：底部 2px 线指示

## 六、性能优化策略

### 6.1 加载优化

```
1. 关键 CSS 内联 ──────── <style> in <head>
   ├── Hero 样式
   ├── 导航样式
   └── 首屏布局
   
2. 非关键 CSS 延迟 ──────── <link rel="preload">
   ├── 卡片样式
   ├── 动画样式
   
3. 字体预加载 ──────── <link rel="preconnect">
   ├── Google Fonts CDN
   └── font-display: swap
   
4. 图片优化 ────────
   ├── WebP 格式
   ├── lazy loading
   ├── placeholder 占位
   ├── 尺寸响应式
```

### 6.2 渲染优化

```
1. 避免布局抖动 ────────
   ├── 固定尺寸容器
   ├── 预留图片空间
   └── transform 替代 top/left
   
2. GPU 加速 ────────
   ├── transform 动画
   ├── will-change 提示
   └── 限制动画元素数量
   
3. 滚动优化 ────────
   ├── Intersection Observer
   ├── 虚拟滚动（如需要）
   └── passive: true
```

### 6.3 性能目标

| 指标 | 目标值 |
|------|--------|
| 首屏渲染 (FCP) | < 1s |
| 可交互时间 (TTI) | < 1.5s |
| 最大内容渲染 (LCP) | < 2s |
| 累积布局偏移 (CLS) | < 0.1 |
| 总阻塞时间 (TBT) | < 200ms |

## 七、无障碍设计

### 7.1 视觉无障碍

- **对比度**：正文 ≥ 4.5:1，标题 ≥ 3:1
- **字体大小**：正文 ≥ 16px，支持 200% 放大
- **色彩不唯一**：状态信息不只依赖颜色

### 7.2 交互无障碍

- **键盘导航**：所有交互元素可访问
- **焦点状态**：清晰的 focus 样式
- **跳转链接**：Skip to content

### 7.3 内容无障碍

- **语义化 HTML**：正确使用 heading、main、nav
- **ARIA 标签**：必要的 aria-label
- **Alt 文本**：所有图片有描述

### 9.1 内容决策

| 问题 | 选项 | 建议 |

| 问题 | 选项 | 建议 |

| 问题 | 选项 | 建议 |

| **页面数量** | 单页 vs 多页 | 单页（展示美学） |
| **暗色模式** | 支持 vs 不支持 | 暂不支持（专注单一体验） |
